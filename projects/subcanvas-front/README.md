1. 프로젝트 개요

	•	프로젝트명: 서브캔버스 (SubCanvas)
	•	슬로건: 당신의 모든 것을 담는 단 하나의 링크
	•	서비스 요약: 사용자가 플러터(Flutter) 앱을 통해 자신의 사진, 소개, SNS 링크 등을 담은 프로필을 만들면, 고유한 웹 URL이 즉시 생성되는 서비스. 사용자는 텍스트로 원하는 디자인을 설명하여 개성을 표현할 수 있으며, 생성된 QR코드를 통해 손쉽게 자신의 웹 프로필을 공유하고 홍보할 수 있다.

2. 서비스 목표

	•	쉬운 제작: 누구나 몇 분 안에 자신만의 웹 프로필을 제작할 수 있는 간편한 경험을 제공한다.
	•	개성 표현: 정형화된 SNS 프로필에서 벗어나, 디자인 설명 기능을 통해 자신만의 아이덴티티와 브랜드를 자유롭게 표현할 수 있는 공간을 제공한다.
	•	간편한 공유: 생성된 QR코드와 고유 URL을 통해 온/오프라인 어디서든 자신을 효과적으로 소개할 수 있는 디지털 명함 역할을 수행한다.
	•	연결의 확장: 여러 SNS와 활동 채널을 하나의 페이지에 모아 보여줌으로써 사용자의 다양한 활동을 통합적으로 알리는 허브 역할을 한다.

3. 타겟 사용자

	•	크리에이터/아티스트/프리랜서: 자신의 포트폴리오나 작업물을 여러 채널에 흩어져 있는 링크 대신 하나의 페이지로 깔끔하게 보여주고 싶은 분
	•	대학생 및 취업준비생: 대외활동, 프로젝트, 개인 브랜딩을 위한 간단한 자기소개 페이지가 필요한 분
	•	네트워킹이 잦은 직장인: 명함을 대체할 수 있는 독특하고 기억에 남는 디지털 프로필을 원하는 분
	•	Z세대 사용자: 자신만의 개성을 독창적인 방식으로 표현하고 공유하고 싶은 모든 분

## 3. 핵심 동작 원리 (웹페이지는 어떻게 생성되는가?)

가장 궁금해하시는 "Generate 버튼을 누르면 어떻게 웹페이지가 만들어지는가?"에 대한 답변입니다.
중요한 점: 실제로 사용자가 버튼을 누를 때마다 my-profile.html 같은 물리적인 파일을 하나씩 생성하는 것이 아닙니다. 그렇게 하면 수만 개의 파일이 생겨 비효율적입니다. 대신, '틀'을 하나 만들어두고 요청이 올 때마다 내용물만 바꿔서 보여주는 방식을 사용합니다.

Step 1: 정보 저장 (앱 → 서버)

	0.	사용자가 앱에서 사진, 자기소개, 고유 주소(my-profile), 디자인 설명(어둡게)을 입력하고 'Generate' 버튼을 누릅니다.
	0.	플러터 앱은 이 모든 정보를 하나의 데이터 묶음(JSON 형태)으로 만들어 백엔드 서버에 전송(HTTP POST 요청)합니다.
	0.	서버는 받은 데이터를 데이터베이스(DB)에 저장합니다. 이때 my-profile이라는 고유 주소가 핵심 식별자(Key)가 됩니다.
DB 저장 예시 (Table: Profiles)

["img1.jpg", "img2.jpg"]

"어둡게"
Sheets로 내보내기

Step 2: 페이지 요청 및 동적 생성 (웹 브라우저 → 서버 → 웹 브라우저)

	0.	다른 사람이 https://subcanvas.com/my-profile 이라는 주소로 접속합니다.
	0.	웹 브라우저는 subcanvas.com 서버에게 "**"my-profile"**에 해당하는 페이지를 보여줘!" 라고 요청합니다.
	0.	서버의 역할: a. 요청받은 주소의 맨 끝부분, 즉 my-profile 이라는 값을 가져옵니다. b. DB에서 unique_id가 my-profile인 사용자의 데이터를 찾습니다. c. 미리 만들어 둔 웹페이지 '틀'(Template)을 가져옵니다. d. '틀'의 비어있는 공간(이름, 자기소개, 사진 주소 등)에 DB에서 찾아온 데이터를 채워 넣습니다. e. 사용자의 '디자인 설명'(어둡게)을 분석하여, 미리 만들어둔 **'어두운 테마 CSS'**를 이 페이지에 적용합니다. f. 데이터와 디자인이 모두 적용된 완성된 HTML 코드를 웹 브라우저에게 응답으로 보내줍니다.
	0.	웹 브라우저는 서버로부터 받은 HTML 코드를 화면에 그려줍니다.
이 모든 과정은 눈 깜짝할 사이에 일어납니다. 이 방식을 **'서버 사이드 렌더링(Server-Side Rendering, SSR)'**이라고 부릅니다.



## 4. 디자인은 어떻게 구성하는가? (Default vs Custom)

디자인 설명에 따라 다른 결과물을 보여주는 기능은 **'조건부 스타일링'**으로 구현합니다. 서버는 미리 여러 벌의 옷(CSS 파일 또는 클래스)을 준비해두고, 사용자의 요청에 맞는 옷을 입혀주는 것과 같습니다.

1. 기본(Default) 디자인: '몽글몽글' 스타일

사용자가 디자인 설명을 비워두면, 서버는 기본값으로 default-theme.css를 적용합니다.
default-theme.css 예시:
CSS

body {
  background-color: #FFFAF0; /* 부드러운 아이보리 배경 */
  font-family: 'Gamja Flower', cursive; /* 귀여운 손글씨 폰트 */
}
.profile-card {
  background-color: white;
  border-radius: 25px; /* 둥글둥글한 모서리 */
  box-shadow: 0 10px 20px rgba(0,0,0,0.1); /* 부드러운 그림자 */
}
.sns-icon {
  transform: scale(1.0);
  transition: transform 0.2s;
}
.sns-icon:hover {
  transform: scale(1.1); /* 마우스를 올리면 몽글하게 커지는 효과 */
}

2. 커스텀 디자인: '어둡고 날카로운' 스타일

사용자가 "어두운 분위기에 날카로운 디자인. SNS 아이콘은 오른쪽 하단 위치." 라고 입력했을 경우, 서버는 이 텍스트에서 키워드를 분석합니다.

미리 디자인 조각(CSS)을 만들어 둡니다.
	•	배경 테마: .theme-dark (어두운 배경), .theme-light (밝은 배경), .theme-pastel (파스텔톤 배경)
	•	모서리 스타일: .style-sharp (각진 모서리), .style-rounded (둥근 모서리)
	•	폰트 스타일: .font-gothic (고딕체), .font-serif (명조체)
	•	요소 위치: .sns-bottom-right (SNS 아이콘 오른쪽 아래)
사용자가 컨셉을 텍스트로 입력합니다.
	•	예: "어두운 분위기에 날카로운 디자인. SNS 아이콘은 오른쪽 하단 위치."
서버가 텍스트에서 키워드를 찾아냅니다.
	•	"어두운" → .theme-dark 클래스 선택
	•	"날카로운" → .style-sharp 클래스 선택
	•	"오른쪽 하단" → .sns-bottom-right 클래스 선택
찾아낸 CSS 클래스들을 웹페이지에 적용하여 보여줍니다.
	•	최종적으로 사용자의 프로필 페이지는 <div class="profile-card theme-dark style-sharp"> 와 같은 형태로 구성됩니다.

미리 만드는 css는 최대한 많은 케이스가 필요함.

	•	'어두운' → theme-dark 클래스 적용
	•	'날카로운' → style-sharp 클래스 적용
	•	'오른쪽 하단' → position-bottom-right 클래스 적용
custom-themes.css 예시:
CSS

/* 어두운 테마 */
body.theme-dark {
  background-color: #1a1a1a;
  color: white;
}
.profile-card.theme-dark {
  background-color: #2b2b2b;
}

/* 날카로운 스타일 */
.profile-card.style-sharp {
  border-radius: 0; /* 각진 모서리 */
}

/* SNS 아이콘 위치 */
.sns-container.position-bottom-right {
  position: fixed;
  bottom: 20px;
  right: 20px;
}
서버는 이 클래스들을 웹페이지 '틀'의 해당 요소에 추가하여 최종 HTML을 완성합니다.
